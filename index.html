What is GitHub Actions?

GitHub Actions is a CI/CD platform built into GitHub that automates tasks in your repository — building code, running tests, deploying apps, sending notifications — triggered by events like pushes, pull requests, or schedules.

workflow A YAML file in .github/workflows/ that defines automated tasks.
event Something that triggers a workflow: push, PR, schedule, manual.
job A set of steps that run on the same runner.
step A single task — either a shell command or an action.
action A reusable unit of code, e.g. actions/checkout@v4.
runner The machine (VM or self-hosted) that executes the job.
1.2

Workflow file structure

Every workflow follows the same top-level shape: name, on, jobs.

.github/workflows/skeleton.yml 
name: <workflow name>
on: <trigger>
jobs:
  <job-id>:
    runs-on: <runner>
    steps:
      - name: <step name>
        run: <command>
  • YAML is indentation-sensitive — use 2 spaces, no tabs.
  • Steps inside a job run sequentially, top to bottom, on the same runner.
  • Each job starts with a fresh runner — files don't persist between jobs.
1.3

Triggers — when workflows run

The on: keyword defines what events run a workflow. You can listen to one event or many.

triggers.yml 
# run on push to main
on:
  push:
    branches: [main]

# manual button in Actions tab
on:
  workflow_dispatch:

# scheduled — 2:00 AM UTC daily (= 7:30 AM IST)
on:
  schedule:
    - cron: '0 2 * * *'

# multiple triggers combined
on:
  push: { branches: [main] }
  pull_request: { branches: [main] }
  workflow_dispatch:
gotcha Cron schedules run in UTC, not local time. IST = UTC + 5:30.

My first workflow — running on GitHub's cloud

✓ completed

The first workflow I shipped. It spins up a fresh Ubuntu VM, prints a greeting, the current date, and the runner's OS — then tears the VM down.

.github/workflows/01-hello.yml 
name: Hello World

on:
  push:
    branches:
      - main

jobs:
  greet:
    runs-on: ubuntu-latest
    steps:
      - name: Print greeting
        run: echo "Hello, GitHub Actions!"

      - name: Print current date
        run: date

      - name: Print runner OS
        run: echo "Running on ${{ runner.os }}"
runner output
Run echo "Hello Github Actions"
Hello Github Actions

Run date
Mon May 18 15:37:46 UTC 2026

Run echo "running on Linux"
running on Linux

Cleaning up orphan processes

→ what I noticed

  • Runner timezone is UTC. That 15:37 UTC was 21:07 IST on my clock.
  • ${{ runner.os }} resolved to Linux because runs-on was ubuntu-latest. windows-latest would give Windows.
  • GitHub injects a GITHUB_TOKEN automatically — we'll use it later for API calls and deployments.
  • The VM is destroyed at the end. Anything I want to keep needs artifacts or a registry push.
